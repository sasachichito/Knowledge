# メモ
```
ストーリー集めてプロジェクト全体を見積もる→都度一部の詳細なスケジュールを得る。

【ストーリー集めてプロジェクト全体を見積もる】
会議(スプリント0)を開く。タイミングはプロジェクト開始前。アウトプットはストーリーリストの初版とスケジュール。
顧客の要件となるストーリーを洗い出して優先順位順のリストを作り、ストーリーリストの初版とする。
リストの各ストーリーにストーリーポイントを付与していく。
プロジェクトのリソースを決定する。
いくつかのストーリーの規模を予想する。
ベロシティを予想する。
期間とコストを導出し、全体のスケジュールを作る。
以降、リストに変更を加えたいときは会議(プロダクトバックログリファインメント)を適宜開き、
変更や新規追加されたストーリーのストーリーポイント付与、再スケジューリングまで行う。

【一部の詳細なスケジュールを得る】
実施するタイムボックス（スプリント）の前に計画MTGを開く。
[計画MTG 第一部]
ストーリーリストから、タイムボックスで実現できるだけのストーリーを選択する。
（タイムボックスの長さと、ベロシティから実現可能なストーリーポイント数がわかる）
[計画MTG 第二部]
ストーリーをタスクに分解して理想時間を得る。（本当にタイムボックスで完了できるか確認する）
```
